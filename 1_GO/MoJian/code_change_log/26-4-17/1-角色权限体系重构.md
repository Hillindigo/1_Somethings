# 权限体系重构 — 改动总结

## 权限模型

| 角色 | 文章 | 分类 | 标签 | 评论 | 用户管理 |
|------|------|------|------|------|----------|
| **游客** | 只看已发布 | 只看列表 | 只看列表 | 最多5条 | ❌ |
| **普通用户** | 发布/编辑/删除自己的 | CRUD 全部 | 只看列表 | 发表/删除自己的 | ❌ |
| **管理员** | 全部 | CRUD 全部 | CRUD 全部 | 全部+审核 | 全部 |

## 路由变更对照

| 接口 | 变更前 | 变更后 |
|------|--------|--------|
| `POST /categories` | admin-only (`/admin/categories`) | authenticated (`/categories`) |
| `PUT /categories/:id` | admin-only (`/admin/categories/:id`) | authenticated (`/categories/:id`) |
| `DELETE /categories/:id` | admin-only (`/admin/categories/:id`) | authenticated (`/categories/:id`) |
| `GET /articles/:id/comments` | 公开，无身份识别 | 公开 + `OptionalJWTAuth`，游客限5条 |
| `GET /admin/users` | 不存在 | admin-only，用户列表 |
| `PUT /admin/users/:id/role` | 不存在 | admin-only，更新角色 |
| `DELETE /admin/users/:id` | 不存在 | admin-only，删除用户 |

## 后端改动

### 1. 新增 CLI 创建管理员 — `cmd/createadmin/main.go`
- 通过终端创建管理员账号，用法：
  ```bash
  go run cmd/createadmin/main.go -username admin -password admin123 -email admin@example.com
  ```
- 支持将已有用户升级为管理员

### 2. 用户管理接口 — `internal/handler/user.go`
- `GET /api/v1/admin/users` — 用户列表（分页）
- `PUT /api/v1/admin/users/:id/role` — 更新用户角色
- `DELETE /api/v1/admin/users/:id` — 删除用户

### 3. 分类权限调整 — `internal/router/router.go`
- 分类 CRUD 从 `/admin/categories`（管理员）移至 `/categories`（登录用户）
- 标签管理仍保留在 admin 路由下

### 4. 游客评论限制 — `internal/handler/comment.go`
- 评论列表接口使用 `OptionalJWTAuth()` 中间件识别身份
- 游客最多返回 5 条评论，登录用户无限制

### 5. 新增中间件 — `internal/middleware/auth.go`
- `OptionalJWTAuth()` — 可选 JWT 认证，不强制登录但解析 Token

### 6. Repository/Service 扩展
- `internal/repository/user.go` — 新增 `List`、`Delete` 方法
- `internal/service/user.go` — 新增 `ListUsers`、`UpdateUserRole`、`DeleteUser` 方法

## 前端改动

### 7. 用户管理页面 — `frontend/src/pages/admin/Users.jsx`（新建）
- 用户列表（分页）、角色切换、删除用户

### 8. Dashboard 入口 — `frontend/src/pages/admin/Dashboard.jsx`

### 11. 导航栏 — `frontend/src/components/Navbar.jsx`

### 12. 游客评论限制提示 — `frontend/src/pages/ArticleDetail.jsx`
- 游客看到 5 条评论时，显示提示"登录后可查看全部评论"

## 数据库变更

- `users.role` 字段语义正式化：`0` = 普通用户，`1` = 管理员（原已存在，本次明确语义约束）
- 无新增表、无新增字段、无数据迁移

## 破坏性变更

- **分类 API 路径变更**：`/admin/categories` → `/categories`，所有已有客户端/脚本需同步修改
- **分类 API 权限降级**：从 admin-only 降为 authenticated，管理员专属调用方需注意普通用户也可访问

## 部署注意事项

1. 部署后需执行 CLI 创建首个管理员账号：`go run cmd/createadmin/main.go -username admin -password xxx -email xxx`
2. 前端需重新构建（分类 API 路径已变更）：`npm run build`
3. 无数据库迁移，可直接部署

## 遗留问题

- [x] 管理员可删除自己 / 降级自己，缺少自我保护校验
- [x] 普通用户删除分类会影响其他用户关联的文章，缺少级联保护
- [x] 游客评论限制 5 条为硬编码，后续应改为 `config.yaml` 可配置

## 遗留问题修复（第二轮）

### 13. 管理员自我保护 — `internal/handler/user.go`
- `UpdateUserRole` 增加校验：禁止管理员修改自己的角色（返回 403）
- `DeleteUser` 增加校验：禁止管理员删除自己（返回 403）

### 14. 分类删除级联保护
- `internal/repository/article.go` — 新增 `CountByCategoryID` 方法，统计分类下文章数
- `internal/service/category.go` — `Delete` 方法增加级联保护：分类下有文章时拒绝删除
- `internal/service/category.go` — `NewCategoryService` 新增 `articleRepo` 参数
- `internal/handler/category.go` — `Delete` 区分"分类不存在"（404）和"分类下有文章"（400）
- `pkg/errcode/errcode.go` — 新增 `ErrCategoryHasArticles = 40004`
- `cmd/server/main.go` — 更新 `NewCategoryService` 调用，传入 `articleRepo`

### 15. 游客评论限制可配置
- `internal/config/config.go` — 新增 `BlogConfig` 结构体（含 `GuestCommentLimit`），`Config` 增加 `Blog` 字段
- `config/config.yaml` — 新增 `blog.guest_comment_limit: 5`
- `internal/handler/comment.go` — `ListByArticleID` 从 `config.GlobalConfig.Blog.GuestCommentLimit` 读取限制数；响应改为结构化对象，携带 `comments`、`guest_comment_limit`、`is_limited`
- `frontend/src/pages/ArticleDetail.jsx` — 适配新响应结构，动态显示限制数
