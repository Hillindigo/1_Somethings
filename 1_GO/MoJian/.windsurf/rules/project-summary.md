# 项目规则 — 记录

## 我让你记录的时候，你才开始记录，记录在 code_change_log 文件夹中，按日期和序号分类。

# 项目规则 — 变更日志规范

## 变更日志目录结构

每次重构项目或功能变动时，必须将变更记录写入 `code_change_log` 文件夹，按以下结构组织：

```
code_change_log/
  └── YY-M-D/              # 按天建文件夹，如 26-4-17
        └── 序号-变更描述.md  # 如 1-权限体系重构.md
```

## 命名规则

- **天文件夹**：`YY-M-D` 格式（年份两位，月份不补零，日期不补零），如 `26-4-17`、`26-12-3`
- **变更文件**：`序号-变更描述.md`，序号从 1 开始递增，描述用中文概括本次变更的核心内容
  - 示例：`1-权限体系重构.md`、`2-新增标签管理.md`、`3-修复评论排序.md`
- 同一天多次变更，序号递增：`1-xxx.md`、`2-xxx.md`、`3-xxx.md`

## 变更文件内容格式

每个变更文件必须包含以下结构（按需增减章节）：

```markdown
# 变更标题 — 改动总结

## 变更概览（可选：如权限模型表格等总览信息）

## 路由变更对照（涉及接口变更时必填）

| 接口 | 变更前 | 变更后 |
|------|--------|--------|
| `METHOD /path` | 原权限/路径 | 新权限/路径 |

- 简要描述即可，重点标注权限等级变化（如 admin-only → authenticated）
- 新增接口标注"不存在"
- 删除接口标注"已移除"

## 后端改动

### 1. 变更描述 — `文件路径`
- 具体改动说明
- 新增/修改/删除的接口、方法、字段等

## 前端改动

### N. 变更描述 — `文件路径`
- 具体改动说明

## 数据库变更（涉及模型/表结构变动时必填）

- 字段语义变更、新增表/字段、数据迁移说明
- 无变更时标注"无新增表、无新增字段、无数据迁移"

## 破坏性变更（涉及接口契约/路径变更时必填）

- API 路径变更、权限等级变更、请求/响应格式变更等
- 无破坏性变更时标注"无"

## 部署注意事项（涉及上线操作时必填）

- 部署前/后需执行的命令、配置变更、数据迁移等
- 无特殊操作时标注"无特殊操作，可直接部署"

## 遗留问题（可选）

- [ ] 问题描述（用 checkbox 格式，修复后勾选）

## 测试数据（可选）

相关接口的测试 JSON 数据或命令行示例
```

## 关键要求

- **每次功能变动必须记录**，包括新增、变更、删除功能
- **文件路径必须标注**，使用相对路径（如 `internal/handler/user.go`）
- **接口变更必须列出** HTTP 方法 + 路径 + 说明
- **路由变更对照表必填**，涉及接口权限/路径变更时必须填写变更前 vs 变更后
- **权限变更必须明确**，列出角色权限对照表
- 变更文件使用 **中文** 撰写
