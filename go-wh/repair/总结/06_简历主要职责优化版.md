### 职责 1：仓库管理模块（核心亮点：三级架构 + 权限体系 + 批量查询优化）

**优化前：**
> 仓库管理模块：负责三级仓库体系的设计与实现，包括仓库 CRUD、权限管理、成员管理、树形结构查询等功能

**优化后（基于实际代码）：**
> **仓库管理模块**：设计并实现三级仓库管理体系（洲级-国家级-具体仓库），支持层级化仓库管理。实现基于角色的权限控制（系统管理员/仓库管理员/成员），通过 `user_warehouse` 表管理用户-仓库多对多关系。使用 `BatchGetWarehouseAdminUserIDs` 和 `BatchGetWarehouseMemberUserIDs` 批量查询优化解决 N+1 问题，支持仓库树形结构递归查询和完整路径（FullPath）自动维护

**实际代码依据：**
```go

const (
```

**关键技术点：**
- **三级层级**：洲级（Level 1）→ 国家级（Level 2）→ 具体仓库（Level 3）
- **权限体系**：系统管理员、仓库管理员（admin）、普通成员（member）
- **批量查询**：`BatchGetWarehouseAdminUserIDs`、`BatchGetWarehouseMemberUserIDs` 解决 N+1 问题
- **数据表设计**：`warehouse` 表 + `user_warehouse` 关联表
- **完整路径**：自动维护 `full_path` 字段（如：/亚洲/中国/北京仓库）

### 职责 2：Excel 批量导入模块（核心亮点：异步处理 + 分批导入 + 并发控制）

**优化前：**
> Excel 导入模块：使用 Worker Pool + 批量插入优化 Excel 导入性能，从 500 条/秒提升到 5000 条/秒，支持导入任务异步处理和进度追踪

**优化后（基于实际代码）：**
> **Excel 批量导入模块**：设计异步导入架构，使用自定义 `sideEffectDispatcher`（并发度 8）实现 Worker Pool 模式。采用分批处理策略（每批 1000 条），使用 MongoDB `BulkCreate` 和 `BulkWrite` 批量操作，配合 60 秒超时控制。实现重复数据智能检测、策略模式处理不同导入场景（机房/仓库 × 主机/部件），支持错误容错和进度实时追踪

**实际代码依据：**
```go
// 并发控制器（import_worker_start_import_patterns.go:24-28）
func newSideEffectDispatcher(maxConcurrent int) *sideEffectDispatcher {
    if maxConcurrent <= 0 {
        maxConcurrent = 8  // 默认并发度为 8
    }
    return &sideEffectDispatcher{sem: make(chan struct{}, maxConcurrent)}
}

// 分批处理（import_worker_start_import_patterns.go:91）
limit: 1000,  // 每批处理 1000 条

// 批量写入（import_worker_start_import_patterns.go:247）
err = r.assetStorage.BulkCreate(createCtx, toInsert)

// 超时控制（import_worker_start_import_patterns.go:246-248）
createCtx, cancel := context.WithTimeout(r.bg, 60*time.Second)
err = r.assetStorage.BulkCreate(createCtx, toInsert)
cancel()
```

**关键技术点：**
- **并发控制**：自定义 `sideEffectDispatcher`，并发度固定为 **8**
- **分批处理**：每批 **1000 条**数据（`limit: 1000`）
- **批量操作**：`BulkCreate`（新增）、`BulkWrite`（更新）
- **超时控制**：每个批次操作超时时间 **60 秒**
- **信号量机制**：使用 channel 实现（`sem chan struct{}`）
- **异步处理**：goroutine 后台执行，任务状态机管理
