# 维修保养系统后端服务

**实习时间：** 2025.05 - 2025.09

**技术栈：** Go, GORM, MongoDB, gRPC, Protobuf, Docker, GitLab CI/CD

---

## 项目描述

基于 Go 语言构建的企业级维修保养管理系统后端服务，为全球化维修业务提供完整的后端支持。项目采用 Gin 框架、GORM ORM 和 MongoDB 数据库，并深度整合 gRPC 等技术，实现了资产管理、仓库管理、项目管理、维修记录等核心功能，旨在提供高效、可扩展的维修保养业务平台。

---

## 核心职责与成果

### 仓库管理模块开发

负责仓库管理核心模块的设计与实现，支持全球化三级仓库体系（洲级/国家级/具体仓库），为资产流转提供基础支撑。

- **数据层设计与实现**：
  - 设计并实现 `Warehouse`、`UserWarehouse`、`AssetInOutRecord` 等核心数据模型
  - 使用 MongoDB 实现仓库数据的持久化存储，支持复杂查询和聚合操作
  - 实现仓库层级关系管理，支持父子级联查询和完整路径构建
  - 设计软删除机制，保障数据安全性和可追溯性

- **业务逻辑层开发**：
  - 实现仓库 CRUD 操作，包括创建、查询、更新、删除等基础功能
  - 开发仓库树形结构查询接口，支持前端树形展示需求
  - 实现仓库成员管理功能，支持管理员和普通成员角色分配
  - 开发仓库权限校验机制，确保用户只能操作有权限的仓库
  - 实现仓库排序功能，支持拖拽排序和同级排序

- **资产出入库管理**：
  - 设计并实现资产出入库记录功能，追踪资产在仓库间的流转
  - 支持多种出库去向（仓库/项目/维修中心/厂商）的统一管理
  - 实现出库统计功能，提供仓库出库量和出库去向 Top3 统计
  - 开发批量操作接口，提升资产批量出入库效率

### API 接口设计与实现

使用 gRPC 和 Protobuf 设计并实现 RESTful 风格的 API 接口，涵盖仓库管理的各个方面。

- **接口设计**：
  - 定义 `warehouse.proto`、`user_warehouse.proto`、`ware_asset.proto` 等 Protobuf 文件
  - 设计 20+ 个仓库相关 API 接口，包括基础 CRUD、树形查询、成员管理、权限控制等
  - 使用 gRPC Gateway 实现 gRPC 到 HTTP/JSON 的自动转换

- **接口实现**：
  - 实现 `WarehouseServiceServer` 接口，提供完整的仓库管理服务
  - 开发分页查询、条件过滤、排序等通用功能
  - 实现接口参数校验和错误处理机制
  - 支持用户上下文传递和权限验证

### 数据库设计与优化

针对 MongoDB 数据库进行合理的数据模型设计和查询优化。

- **数据模型设计**：
  - 设计仓库三级层级结构，支持 `full_path` 完整路径查询
  - 设计用户-仓库关联表，支持多对多关系和角色管理
  - 设计资产出入库记录表，支持时间范围查询和统计分析
  - 统一使用软删除标记，避免数据物理删除

- **查询优化**：
  - 实现高效的层级查询算法，支持父子级联查询
  - 使用 MongoDB 聚合管道实现复杂统计查询
  - 设计合理的索引策略，提升查询性能
  - 实现批量查询接口，减少数据库往返次数

### 权限控制与安全

实现基于角色的权限控制体系，保障系统安全性。

- **权限设计**：
  - 实现系统管理员、仓库管理员、普通成员三级权限体系
  - 开发权限校验中间件，统一处理权限验证逻辑
  - 支持用户在不同仓库拥有不同角色
  - 实现操作日志记录，追踪用户操作行为

- **数据安全**：
  - 实现软删除机制，防止数据误删
  - 记录创建人、更新人、删除人等审计信息
  - 使用时间戳记录所有数据变更时间
  - 实现修改历史记录功能，支持数据变更追溯

### 代码质量与工程化

遵循 Go 语言最佳实践，保证代码质量和可维护性。

- **代码规范**：
  - 遵循 Go 语言编码规范，使用清晰的命名和注释
  - 实现接口与实现分离，提高代码可测试性
  - 使用依赖注入模式，降低模块耦合度
  - 编写单元测试，保证核心功能的正确性

- **工程化实践**：
  - 使用 GitLab CI/CD 实现自动化构建和部署
  - 使用 Docker 容器化部署，提高部署效率
  - 使用 Makefile 管理项目构建流程
  - 集成代码质量检查工具，保证代码质量

## 技术亮点

1. **三级仓库层级设计**：设计并实现了洲级-国家级-具体仓库的三级层级体系，支持全球化业务需求，通过 `full_path` 字段实现高效的层级查询。

2. **灵活的权限控制**：实现了基于角色的权限控制体系，支持用户在不同仓库拥有不同角色，既保证了数据安全，又提供了灵活的权限管理。

3. **完善的审计机制**：通过软删除、操作日志、修改历史等机制，实现了完整的数据变更追溯，满足企业级应用的审计需求。

4. **高性能查询优化**：使用 MongoDB 聚合管道实现复杂统计查询，通过批量查询接口减少数据库往返，显著提升系统性能。

5. **gRPC 微服务架构**：采用 gRPC 和 Protobuf 实现服务间通信，通过 gRPC Gateway 同时支持 HTTP/JSON 和 gRPC 协议，提高系统的互操作性。
